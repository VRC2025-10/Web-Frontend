# 用語集

> **ナビゲーション**: [ドキュメントホーム](../README.md) > [リファレンス](README.md) > 用語集

本プロジェクトで使用される用語の定義集です。

## ドメイン用語

| 用語 | 英語表記 | 定義 |
|------|---------|------|
| 同期会 | Class Reunion | 本システムが管理する同窓会組織。メンバーの管理、イベントの開催、部活動の運営などを行う中心的な単位 |
| メンバー | Member | 同期会に所属するユーザー。Discord サーバーへの参加と OAuth2 認証により登録される |
| プロフィール | Profile | メンバーの公開情報（表示名、自己紹介、卒業年度など）。各メンバーにつき1つ作成できる |
| 部活動 | Club | メンバーが自主的に結成する活動グループ。趣味や興味に基づくコミュニティ単位 |
| ギャラリー | Gallery | メンバーがアップロードした画像の集合。プロフィールや部活動に紐づけて管理される |
| イベント | Event | 同期会が主催する集まりや活動。日時・場所・参加者の管理機能を提供する |
| イベントタグ | Event Tag | イベントに付与する分類用ラベル。カテゴリ別のフィルタリングに使用される |

## API 種別

| 用語 | 定義 |
|------|------|
| System API | GAS（Google Apps Script）などの外部システムから呼び出される API。`X-System-Token` ヘッダーによるトークン認証で保護される |
| Internal API | 認証済みメンバーが使用する API。セッション Cookie による認証が必要。ロールに応じたアクセス制御が適用される |
| Public API | 認証不要で誰でもアクセスできる API。公開情報の閲覧のみ提供する |
| Auth API | Discord OAuth2 認証フローを管理するエンドポイント群。ログイン開始・コールバック処理・ログアウトを提供する |

## 外部サービス・インフラ

| 用語 | 英語表記 | 定義 |
|------|---------|------|
| GAS | Google Apps Script | Google スプレッドシートと連携してメンバーデータの一括同期を行う外部スクリプト。System API 経由でバックエンドと通信する |
| ギルド | Guild（Discord サーバー） | Discord における「サーバー」の API 上の正式名称。本システムでは同期会が運用する Discord サーバーを指す |
| Caddy | — | 本番環境で使用するリバースプロキシサーバー。自動 HTTPS 証明書管理と HTTP/2 対応を提供する |

## 認証・セキュリティ

| 用語 | 定義 |
|------|------|
| セッション | サーバー側に保存されるユーザーの認証状態。PostgreSQL をバックエンドストアとして使用し、Cookie の `session_id` と紐づけて管理する |
| OAuth State | Discord OAuth2 認証フローにおける CSRF 防止用パラメータ。認証リクエスト時に生成され、コールバック時に検証される |
| RBAC | Role-Based Access Control（ロールベースアクセス制御）。ユーザーに割り当てられたロール（member, admin, super_admin）に基づいてアクセス権を判定する仕組み |
| CSRF トークン | Cross-Site Request Forgery（クロスサイトリクエストフォージェリ）対策用のトークン。状態変更を伴うリクエストをブラウザが正当に発行したものか検証するために使用する |

## アーキテクチャ

| 用語 | 定義 |
|------|------|
| ヘキサゴナルアーキテクチャ | ビジネスロジック（ドメイン層）を外部依存から隔離する設計パターン。ポートとアダプターのパターンとも呼ばれる。本システムの基本設計方針 |
| ポート | ヘキサゴナルアーキテクチャにおけるインターフェース定義。ドメイン層が外部とやり取りするための抽象化された境界。Rust ではトレイトとして定義する |
| アダプター | ポート（トレイト）の具体的な実装。データベースアクセス、外部 API 呼び出しなどの具体的な処理を担当する |
| Tower Layer | axum のミドルウェアフレームワーク。リクエストの前後処理（認証チェック、ログ記録、レート制限など）を再利用可能な層として実装する仕組み |

## Rust 固有の概念

| 用語 | 定義 |
|------|------|
| Type-State パターン | 型システムを利用して状態遷移を静的に保証する設計パターン。不正な状態遷移をコンパイル時に検出できる。本システムでは OAuth2 フローの状態管理などに使用 |
| コンパイル時検証 | Rust の型システムとマクロを活用し、実行時ではなくコンパイル時にエラーを検出する手法。SQL クエリの検証（sqlx）など、多くの場面で活用している |
| エラー代数 | `enum` によるエラー型の定義パターン。全てのエラーケースを網羅的に列挙し、パターンマッチングで安全にハンドリングする。本システムでは `DomainError` として実装 |
