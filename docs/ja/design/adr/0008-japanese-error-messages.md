# ADR-0008: 日本語エラーメッセージ

> **ナビゲーション**: [ドキュメントホーム](../../README.md) > [設計](../README.md) > [ADR](README.md) > ADR-0008

## ステータス

**承認済み**

## 日付

2025-01-10

## コンテキスト

VRC Web-Backend は主に日本語話者のコミュニティにサービスを提供します。ユーザーがエラー（バリデーション失敗、認可問題等）に遭遇した場合、エラーメッセージはフロントエンド UI に表示されます。

### 影響する力

- 主要な対象者は日本語話者
- i18n の導入はフレームワーク依存と複雑さを増す
- エラーコード（例: `ERR-AUTH-001`）が言語非依存の機械可読識別子を提供
- コードベースと内部ドキュメントは英語
- コミュニティは近い将来国際展開する可能性は低い

## 決定

**すべてのユーザー向けエラーメッセージに日本語を使用**します。プログラム的ハンドリングのためにエラーコードを併記します。

### 実装

```rust
impl From<DomainError> for ApiError {
    fn from(err: DomainError) -> Self {
        match err {
            DomainError::UserNotFound => ApiError::NotFound {
                code: "ERR-USR-001",
                message: "ユーザーが見つかりません".into(),
            },
            DomainError::UserSuspended => ApiError::Forbidden {
                code: "ERR-USR-002",
                message: "アカウントが停止されています".into(),
            },
        }
    }
}
```

### レスポンスフォーマット

```json
{
  "error": {
    "code": "ERR-USR-001",
    "message": "ユーザーが見つかりません"
  }
}
```

### 言語境界

| コンテキスト | 言語 |
|------------|------|
| ユーザー向けエラーメッセージ | 日本語 |
| エラーコード | 英語（機械可読） |
| ログメッセージ | 英語 |
| コードコメント | 英語 |
| 内部ドキュメント | 英語 |
| API ドキュメント | 英語（日本語の例付き） |

## 結果

### ポジティブ

- ネイティブ言語でのユーザー体験
- i18n フレームワーク不要（単一言語）
- エラーコードがプログラム的ハンドリングを提供

### ネガティブ

- 日本語を読めない開発者がメッセージを理解できない
- 後からの i18n 追加はメッセージ抽出が必要
