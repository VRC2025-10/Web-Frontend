## ADR とは

アーキテクチャ決定記録は、重要なアーキテクチャ決定をそのコンテキストと結果とともに記録します。各 ADR は単一の決定、その理由、受け入れたトレードオフを記述します。

[ADR テンプレート](template.md)に記述されたフォーマットに従います。

## ステータス定義

| ステータス | 意味 |
|----------|------|
| ✅ 承認済み | 決定が有効であり実装済み |
| 🔄 提案中 | 決定が議論中 |
| ⛔ 非推奨 | 決定が新しい ADR に取って代わられた |
| ❌ 却下 | 決定が検討されたが採用されなかった |

## 新しい ADR の作成

1. [テンプレート](template.md)をコピー
2. 連番で番号付け（例: `0009-my-decision.md`）
3. すべてのセクションを記入 — コンテキストが最も重要
4. ステータスを**提案中**に設定
5. レビュー用の PR をオープン
6. マージ時にステータスを**承認済み**に更新
