# ADR-0005: Kani による形式検証

> **ナビゲーション**: [ドキュメントホーム](../../README.md) > [設計](../README.md) > [ADR](README.md) > ADR-0005

## ステータス

**承認済み**

## 日付

2025-02-01

## コンテキスト

VRC Web-Backend にはすべての条件下で保持されなければならない重要なドメイン不変条件があります:
- ロール階層は推移的でなければならない
- サニタイズされたコンテンツにスクリプトタグが含まれてはならない
- レート制限カウンターがオーバーフローしてはならない

例示ベーステストは特定の入力を検証しますが、**すべて**の入力に対してプロパティが成り立つことを証明できません。

## 決定

重要なドメイン不変条件を形式的に検証するために **Kani 有界モデル検査**を使用します。

### Kani が行うこと

Rust コードを数学モデルに変換し、有界入力内のすべての可能な実行パスを網羅的にチェックします。

### 適用箇所

| 不変条件 | 重要性 |
|---------|--------|
| ロール階層の推移性 | 認可の正しさ |
| サニタイズの完全性 | XSS 防止 |
| カウンターオーバーフロー安全性 | レート制限の正しさ |
| トークン比較の定時間性 | タイミング攻撃防止 |

## 結果

### ポジティブ

- テスト入力だけでなく有界範囲内のすべての入力に対する数学的保証
- 実際の Rust コードで動作（別の仕様言語不要）
- 教育的価値

### ネガティブ

- 追加のツールチェーンが必要
- 証明は有界（無界保証ではない）
- 実行時間が長い（ナイトリー CI で実行）
